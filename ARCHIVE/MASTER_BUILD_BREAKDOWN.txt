Dark Holds Master Build Breakdown
=================================

Files in this folder
--------------------
1. DarkHoldsFloor1PrototypeMasterBuild.java
   - Single-file master source build.
   - This merges the current modular Floor 1 room structure, the item bank, and the bestiary bank into one compile-ready file.
2. MASTER_BUILD_BREAKDOWN.txt

Compile / Run
-------------
Compile:
  javac DarkHoldsFloor1PrototypeMasterBuild.java
Run:
  java DarkHoldsFloor1PrototypeMasterBuild

Current build overview
----------------------
This build keeps the newer modular Floor 1 structure and merges all support code into one file.
It is still a prototype build, so several systems and content entries exist in code before they are fully placed into rooms.

Current Floor 1 structure in use
--------------------------------
The current floor generator uses the newer modular 6-room skeleton:
- f1safe: Safe Room
- r2: one random puzzle-slot room
  - Broken Fork OR Serpent Shrine
- r3: one random fight-slot room
  - Bat Roost OR Goblin Trail OR Damp Tunnel
- f1lockedHall: Locked Hallway
- r5: one random remaining-pool room
  - leftover puzzle room, leftover fight room, Empty Cache, or Snake Hoard
- f1snakeBoss: Serpent Lair

Rewards currently placed by code
--------------------------------
- Iron Key: placed in one key-eligible room
- Torch: placed in one pre-lock room other than Safe Room
- Small Potion x3: placed in up to three pre-lock rooms
- Ancient Relic: placed on the boss room in this build
- Extra boss gold: 25 to 50 gold on the boss room

Combat systems currently in use
-------------------------------
- Basic damage formula:
  player damage = max(1, movePower + playerAttack - enemyDefense)
  enemy damage  = max(1, movePower + enemyAttack - playerDefense)
- Pocket Sand still always does 1 damage on hit.
- Constrict still applies the snake constrict state and follow-up damage.
- Dodge is active in battle.
- Parry is coded and active only if the player has an item that grants parry.
- Most physical attacks are marked parryable.
- Light, status, and magic attacks are usually not parryable.
- Constrict is explicitly not parryable.

What is in the code but not actively used yet
---------------------------------------------
1. Advanced enemy roster not yet placed into Floor 1
   - Wolf, Skeleton, Orc, Ogre, Cave Troll, Necromancer all exist, but the current Floor 1 generator does not spawn them.

2. Several items are defined but not currently awarded or sold
   - Sword
   - Big Potion
   - Scroll of Escape
   - Scroll of Fireball
   - Scroll of Stealth
   - Armor
   - Shield
   - Book of Spells

3. Parry is coded, but not showcased by current loot
   - The battle system supports parry.
   - The player only gets access to parry if carrying an item that grants parry.
   - Shield grants parry, but the current Floor 1 build does not place a shield.

4. Scroll of Stealth support is incomplete
   - ItemContext tracks stealth state and encounter bypass flags.
   - The game loop does not currently check those flags to skip encounters.
   - Result: Scroll of Stealth exists as designed content, but is not fully wired into room/battle flow yet.

5. Dark-room support is not active yet
   - TorchItem defines allowsDarkRoomEntry().
   - No room logic currently checks for dark-room access, so this utility flag is unused.

6. Book of Spells is a placeholder
   - It can store spell names in an ArrayList.
   - No gameplay system currently reads or uses stored spells.

7. Shield helper methods are mostly legacy / optional right now
   - ShieldItem still has helper methods like enablesParry(), getBattleActionName(), parryDamage(), and getParryResultMessage().
   - The actual battle loop now uses the generic GameItem.grantsParry() flag and EnemyMove.isParryable().
   - Result: those shield-specific helper methods may be removable later unless the team wants to reuse them.

8. Puzzle rooms are label-only for now
   - Broken Fork and Serpent Shrine exist as puzzle-slot rooms in Floor 1.
   - They currently provide flavor text only.
   - No true puzzle or riddle resolution logic is implemented yet.

9. Snake Hoard is no longer a true post-boss room in this build
   - In the current modular 6-room structure, Snake Hoard can appear as a general-pool room before the boss.
   - It is not currently used as a separate post-boss payout room.
   - Boss rewards are attached directly to the boss room instead.

10. Floor 2 is not present in code here
   - This master build contains only the current Floor 1 prototype.
   - Future floor work still needs to be added later.

Possible clean-up targets at the end of the project
---------------------------------------------------
These are candidates to remove if the team decides not to use them:
- Scroll of Stealth support fields in ItemContext if stealth never gets wired in
- Shield helper methods if the generic parry system replaces them completely
- Book of Spells if no spell-inventory system is built
- Dark-room utility hooks if no dark rooms are added
- Unused enemy factory methods if the final game scope stays smaller
- Puzzle-slot labeling if no puzzle mechanics are added
